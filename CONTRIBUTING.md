# Contributing

## Documentation Standards

Every page must follow this structure:

- Overview
- Purpose
- Business Problem
- Architecture
- Workflow
- Inputs
- Outputs
- AI Model
- Prompt
- JSON Schema
- Database
- API
- Caching
- Retry Logic
- Cost
- Testing
- Future Improvements
- Architecture Decision Record (ADR)

## AI Coding Agents

Before implementing any feature:

1. Read the relevant architecture page
2. Review the corresponding ADR
3. Follow the documented naming conventions
4. Do not invent database fields
5. Do not modify unrelated components
6. Implement exactly as specified
7. Write tests before marking the feature complete

## Pull Requests

- Keep changes focused on a single component
- Update documentation alongside code
- Include evidence of testing
